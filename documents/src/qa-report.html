1. Summary

PENUMBRA v1.0 was verified against the SRS acceptance criteria. The automated suite (35 tests, 35 passed, 0 failed, ~0.8 s) covers the F10.7 forecast, Kp categorical forecast, the leakage-free calibration backtest, the drag translation, and the full API contract. The UI builds clean under strict TypeScript and renders the fan chart, Kp strip, drag panel, and calibration diagrams correctly. Live NOAA connectivity was confirmed. Two findings surfaced during QA (both resolved). Verdict: ready for testing and deployment alongside KESSLER.

4. Findings

IDSeverityFindingDisposition
QA-1MajorSPA catch-all masked API 404s. With ui/dist present, the single-service SPA fallback served index.html (HTTP 200) for unmatched /api/* routes instead of 404.Fixed: the SPA handler now returns the 404 error envelope for api/, docs, openapi.json, redoc prefixes. Regression covered by test_unknown_route_404.
QA-2NoteSkill measured vs persistence, not NOAA historical. NOAA's free endpoint exposes only the current 45-day forecast, not a historical archive, so a reproducible backtest cannot score against past NOAA issues.By design: skill is scored against a reproducible persistence baseline (clearly labelled), and NOAA's live forecast is overlaid on the forecast chart for direct current-issue comparison. Documented in the calibration summary and HLD §3.
QA-3NoteHonest skill dips on demo data. On the strongly-periodic bundled demo series, mean-reversion underperforms persistence at a few leads (visible negative skill).Retained deliberately — showing where the model is weaker is the credibility point; real F10.7 gives a smoother profile.
QA-4NoteDrag translation is a reduced exponential-atmosphere model.Explicitly labelled "reduced-model, order-of-magnitude" in the UI and API (FR-DRAG-3).
